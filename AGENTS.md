# Repository Notes

`tourenbuchctl` is a personal Go CLI for maintaining a Tourenbuch: a digital
and printed logbook for outdoor activities such as mountain biking, ski touring,
and hiking. The CLI scaffolds activities, syncs Strava stats and GPX tracks,
renders maps/elevation profiles, and builds a single-page PDF via Pandoc,
LaTeX, Gnuplot, and local helper scripts.

## Local Data Model

- Activity text files live in a private text repo under
  `/Users/nce/vcs/github/nce/tourenbuch/`.
- Activity assets live in an iCloud-synced folder under
  `/Users/nce/Library/Mobile Documents/com~apple~CloudDocs/privat/sport/Tourenbuch/`.
- Activity directories are usually named `name-DD.MM.YYYY`.
- `input.gpx` is the canonical GPX asset generated from Strava streams.
- `map.png` is a generated Thunderforest-backed route map asset.
- iCloud can cache or delay generated asset updates. If a rendered asset looks
  stale, verify the physical folder being viewed before assuming the renderer is
  wrong.

## Configuration

Configuration is read from `~/.tourenbuchctl` as YAML via Viper. Important keys:

```yaml
STRAVA_CLIENT_ID: "..."
STRAVA_CLIENT_SECRET: "..."
THUNDERFOREST_API_KEY: "..."
activities:
  - mtb
  - skitour
  - wandern
```

Do not commit secrets or print API keys in logs.

## Render Flow

- Strava streams are converted to `input.gpx`.
- `tourenbuchctl gen` extracts GPX data for the elevation profile.
- If `THUNDERFOREST_API_KEY` is configured, `gen` renders `map.png`:
  - Thunderforest raster tiles form the background.
  - `github.com/fogleman/gg` draws the route and direction chevrons.
  - Map style defaults to `cycle` for `mtb`, `outdoors` for `skitour` and
    `wandern`.
- The PDF render uses Pandoc and LaTeX templates from the activity text folder.

Thunderforest/OpenStreetMap attribution must remain visible on exported maps.

## Development

- Prefer small, focused Go packages and existing repo conventions.
- Run `go test ./...` after code changes.
- Use `gofmt` on touched Go files.
- Avoid rewriting user-authored activity files unless explicitly requested.
- Generated map/elevation/PDF artifacts may live outside the repo in the asset
  folder.
- The stravaapi/ folder is autogenerated by swagger and should not be touched
- After feature completion, `task fmt` should run without warnings or errors

## External Documentation

Use the `ctx7` CLI to fetch current documentation whenever the user asks about a
library, framework, SDK, API, CLI tool, or cloud service. Run `ctx7 library`
first to resolve a library ID, then `ctx7 docs`.
